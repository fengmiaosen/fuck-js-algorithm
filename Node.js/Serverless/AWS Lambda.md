
# AWS Lambda 核心概念

> [官方文档](https://aws.amazon.com/cn/lambda/)

AWS Lambda 是一种**无服务器计算服务 (Serverless Computing)**，它允许您在不预置或管理服务器的情况下运行代码。您只需按实际使用的计算时间付费，代码未运行时不产生任何费用。

---

## 主要特点

1.  **事件驱动架构**:
    *   Lambda 函数由**事件**触发，例如：
        *   对 Amazon S3 存储桶的更改。
        *   对 Amazon DynamoDB 表的更新。
        *   通过 Amazon API Gateway 发出的 HTTP 请求。
        *   来自其他 AWS 服务或自定义应用程序的调用。

2.  **自动扩展**:
    *   Lambda 会根据传入请求的数量自动扩展您的应用程序。它可以从每天几个请求扩展到每秒数千个请求，而无需任何手动配置。

3.  **支持多种语言**:
    *   原生支持多种编程语言，包括 **Node.js, Python, Go, Java, C#, 和 Ruby**。
    *   您还可以提供自己的运行时环境 (Custom Runtimes)。

4.  **按使用付费**:
    *   计费基于两个主要指标：**请求数量**和**代码执行持续时间**（从代码开始执行到返回或终止，四舍五入到最接近的 1 毫秒）。

5.  **高可用性和容错性**:
    *   Lambda 在多个可用区 (Availability Zones) 中运行您的代码，以保护单个函数免受数据中心故障的影响。

## 核心优势

*   **无需管理服务器**: 您无需关心操作系统、补丁、扩展或任何其他基础设施管理任务。
*   **持续扩展**: 自动处理容量、扩展、补丁和管理。
*   **成本效益**: 仅在代码运行时付费，非常适合工作负载不均匀或不可预测的应用程序。
*   **快速开发与迭代**: 开发者可以专注于编写业务逻辑代码，而不是管理和部署环境。

## 典型应用场景

*   **Web 应用程序**: 构建无服务器后端，处理 API 请求。
*   **实时文件处理**: 在文件上传到 S3 后立即处理图像、视频或日志。
*   **数据处理**: 运行 ETL (提取、转换、加载) 任务，例如在数据写入数据库时进行验证、排序或筛选。
*   **实时流处理**: 处理来自 Amazon Kinesis 或 DynamoDB Streams 的实时流数据。
*   **聊天机器人和 Alexa Skills**: 为语音助手和聊天机器人提供后端逻辑。
